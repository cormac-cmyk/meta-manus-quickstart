Phase 1
Build Your Meta Developer App
Create your app at Meta for Developers and activate the Marketing API.
0 / 7 ▼
💡 No App Review needed — this is for your own ad account only. You can stay in Development Mode permanently for personal use.
  • ✓
    Go to developers.facebook.com and log in with your Facebook account
  • ✓
    Click "My Apps" → "Create App"
  • ✓
    Under Use Cases, select "Create & manage ads with Marketing API"
    This auto-adds the permissions you need: ads_management, ads_read, business_management.
  • ✓
    Name your app (e.g. Manus Ads Manager), link it to your Business Manager, and click "Create App"
  • ✓
    In the left sidebar, click "Add Product" and add the Marketing API product
    Why? The Marketing API is not active by default. Adding it is what unlocks the ads_management and ads_read endpoints — without this, no API call related to ads will work regardless of what token you have.
  • ✓
    Go to App Settings → Advanced — scroll to "Advertising Accounts" and add your Ad Account ID (numbers only, no act_ prefix)
    Why? This associates your ad account with the app. To manage additional ad accounts (e.g. clients) in the future, you must add each one here — otherwise the API cannot access them through this app.
  • ✓
    Go to App Settings → Basic and copy your App ID and App Secret — save them securely
    Why? Your App ID is required when generating a System User token — Business Manager asks you to select which app the token is for. Your App Secret is used for server-side API verification. Never share either publicly.

Phase 2 — Choose Your Path
Generate Your Access Token
Quick Start for testing (token lasts ~60 min). Advanced for permanent automation.
0 / 0 ▼
⚡
Quick Start
Short-lived token (~60 min). Great for testing. Easiest to set up.
🔐
Advanced — System User
Permanent, non-expiring token. Required for ongoing automation.
⚠️ This token expires in approximately 60 minutes. It's great for testing but you'll need to regenerate it regularly. Use the Advanced path for permanent automation.
  • ✓
    Go to developers.facebook.com/tools/explorer/ (Graph API Explorer)
  • ✓
    Select your app from the dropdown at the top right
  • ✓
    Click "Generate Access Token" — grant the permissions when prompted
    Make sure ads_management, ads_read, and business_management are selected.
  • ✓
    Copy the token — you'll paste this into Manus in Phase 3
🔗 Quick Links
Graph API Explorer
🔐 System User tokens never expire. This is the recommended setup for permanent, uninterrupted Manus automation.
  • ✓
    Go to business.facebook.com → Business Settings → Users → System Users
  • ✓
    Click "Add" → name it (e.g. Manus System User) → set role to Admin → click "Create System User"
    Admin role is required for full ads management access including creating and pausing ads.
  • ✓
    Select your System User → click "Add Assets" → choose "Apps" → select your Meta Developer App → enable "Manage App" → click "Save"
    Critical step: The app must be installed on the System User before you can generate a token for it. If you skip this, the token generation dropdown will not show your app.
  • ✓
    Click "Add Assets" again → choose "Ad Accounts" → select your Ad Account → enable "Manage Ad Account" → click "Save"
    This gives the System User token full read and write access to your ad account. For each additional ad account you want Manus to manage, repeat this step.
  • ✓
    Click "Generate New Token" → select your app from the dropdown → tick all required permissions → set expiry to "Never" → click "Generate Token"
    Required permissions: ads_management, ads_read, business_management, pages_show_list, pages_read_engagement, read_insights.
  • ✓
    Copy and save your token immediately — Meta will not show it again after you close this window
    Store it in a password manager. You will paste this into the Manus Custom MCP configuration in Phase 3.

3
Phase 3
Connect Meta Ads in Manus
Add your token to Manus and start managing ads with AI prompts.
0 / 10 ▼
There are two layers to this connection — you need both
📊 Layer 1 — MCP Connector
Connects Manus to the Meta Marketing API for read access — reports, insights, campaign data. Token stored securely in Manus.
✓ Reports  ✓ Insights  ✓ Data
⚡ Layer 2 — Graph API (System User)
Your System User token lets Manus call the Meta Graph API directly to write actions — launch ads, pause ads, adjust budgets, upload creatives.
✓ Launch  ✓ Pause  ✓ Adjust
Both use the same System User token. The MCP uses it for structured read queries. The Graph API uses it for direct write actions via Python scripts Manus executes on your behalf.
Option A — Built-in Connector (Quick Start token)
  • ✓
    Log in to manus.im → click your profile icon → Settings → Connectors
  • ✓
    Find "Meta Ads Manager" and click "Connect" — log in with Facebook and grant permissions
    Select the Ad Account(s) you want Manus to access when prompted.
  • ✓
    Confirm the connector shows as Connected ✓
Option B — Custom MCP (System User token — permanent)
🔐 Use this if you generated a System User token in Phase 2. This connects Manus directly to the Meta Marketing API — no OAuth, no re-authentication, no expiry.
What to fill in the MCP Configuration Modal
Server Name
Any name, e.g. Meta Ads
Transport Type
Select HTTP
Server URL
https://mcp.pipeboard.co/meta-ads-mcp
Custom Headers
Click "+ Add custom header"
Name: X-META-ACCESS-TOKEN  |  Value: your System User token
There is no separate token field — this custom header IS how the token is passed.
  • ✓
    In Manus, click your profile icon → Settings → Connectors
  • ✓
    Click "+ Add Connector" then select "Custom MCP"
  • ✓
    Set Server Name to anything (e.g. Meta Ads), set Transport Type to HTTP
  • ✓
    Paste into Server URL: https://mcp.pipeboard.co/meta-ads-mcp
  • ✓
    Click "+ Add custom header" — set name to X-META-ACCESS-TOKEN, paste your System User token as the value
    This is where your token goes. There is no separate token field — the custom header is the token field.
  • ✓
    Click "Save" — the MCP server is now active and permanently connected
Test Your Connection
  • ✓
    Start a new Manus task and type: "Show me all active campaigns in my Meta Ads account"
    Manus should return your campaign list with status and spend data.
  • ✓
    Test a write action: "Pause ad [name of an ad]" — confirm Manus can take action, not just read
    Only do this on an ad you're comfortable pausing temporarily.
